# Golden Goose PDA

A high-performance, modular Point-of-Sale (POS) and order management system designed for the hospitality industry.

Built in Kotlin and deployed on Raspberry Pi infrastructure, the system operates as a local low-latency server, enabling real-time order processing, table management, and cashier tracking for restaurants and cafés.

---

## 🚀 Overview

Golden Goose PDA is a complete restaurant operations system focused on:

- Real-time order management
- Table status visualization
- Cashier/session tracking
- Low-latency local server architecture
- High reliability in offline/local environments

The system is optimized for speed, usability, and operational clarity in fast-paced hospitality environments.

---

## 🏗️ System Architecture

The application is built with a modular Kotlin-based architecture composed of multiple interconnected services and classes.

Key components include:

- Core POS engine (order processing logic)
- Table management system
- User/session management (waiters, cashiers)
- Local server layer running on Raspberry Pi
- Data synchronization and state management modules

The system is designed to run fully locally, minimizing dependency on external cloud services.

## 🧠 Design Philosophy

This system is built around one core principle:

> “Speed and clarity in real-world restaurant operations.”

To achieve this, the architecture prioritizes:

- Local-first execution (no cloud dependency for core logic)
- Minimal latency in order processing
- Highly modular and maintainable Kotlin structure
- Direct mapping between real-world restaurant flow and software logic
